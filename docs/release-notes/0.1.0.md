# OnlyCue 0.1.0 — initial MVP release

The first shippable build of OnlyCue. A native macOS cue-list editor for lighting designers and show programmers — import media, lay out cues against the playhead, save as `.cuelist`, reopen later.

## What's in this release

- **Document workflow** — `.cuelist` JSON documents via `DocumentGroup`. Standard ⌘N / ⌘O / ⌘S / window-management behaviors. Saved documents reopen with their media via security-scoped bookmarks.
- **Media import** — drag-drop or ⌘O any AVFoundation-supported audio or video file (`.mp3`, `.wav`, `.aac`, `.m4a`, `.aiff`, `.mp4`, `.mov`, and friends) into a document. Unsupported types are rejected with a clear alert.
- **Preview** — video frames render through `AVPlayerLayer`; audio renders as a peak waveform with on-disk caching keyed by SHA-256 of the source.
- **Transport** — Space toggles play/pause, ←/→ jump 1s, scrub-by-timestamp via cue selection. HH:MM:SS.mmm time readout.
- **Cue list** — right-side inspector showing every cue with a colored swatch, name, and timestamp. Click to seek. Empty state hints discoverability of `M`.
- **Cue editing** — `M` adds a cue at the current playhead. Double-click to rename inline. Click the swatch for an 8-color palette popover. ⌫ deletes the selected row. Every mutation flows through `CueCommands` and is undoable via ⌘Z / ⌘⇧Z, with named Edit-menu actions.
- **Cue markers on the waveform** — colored caps + vertical lines drawn at each cue's timestamp. Tap a marker to seek; drag it to retime as a single undo step.
- **Polish** — empty document hint, navigation subtitle showing the imported file name, About panel with version + credits, first-launch welcome sheet, placeholder app icon. Missing-media on document reopen surfaces a "Relink media…" alert that refreshes the bookmark and reloads playback.

## Install

1. Download `OnlyCue-0.1.0.dmg` below.
2. Open the DMG and drag **OnlyCue** into your Applications folder.
3. Eject the DMG.
4. **First launch:** in Finder, right-click (or Control-click) `OnlyCue.app` and choose **Open**. macOS will warn that the developer can't be verified — click **Open** anyway. Future launches are silent.

Why the right-click step? This release is distributed without a paid Apple Developer ID signature. The `.app` is ad-hoc signed and unmodified; right-clicking → Open is the standard macOS bypass for Gatekeeper. We may upgrade to Developer ID + notarization in a later release.

System requirements: macOS 14 (Sonoma) or later. Apple silicon and Intel both supported.

Phase 2 explores LTC timecode generation, cue templates, CSV / Resolve EDL export, OSC / MIDI integrations, and beat-detection-assisted cue placement. See [`docs/roadmap.md`](https://github.com/chienchuanw/only-cue/blob/main/docs/roadmap.md).

## Known limitations

- Gatekeeper shows a "developer cannot be verified" prompt on first launch (free-tier distribution).
- App Sandbox is off.
- No auto-update yet (Sparkle is post-MVP).
- Cue list selection responds to ←/→ for row navigation, which can compete with the global ←/→ "jump 1s" shortcut when the inspector is focused. Click the main pane to give arrow keys back to the transport.
